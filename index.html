    • This is my notebook for the Primer series at Portland Code School. It collects terms, definitions, journal entries, and at least one picture of my face.

    My Goals

    Being self taught in anything is bound to result in small random gaps in knowledge. My goal for this class is to fill in those gaps. I also need to learn some best practices because it's never easy to know if you're doing something badly if it works.

    I also lack the soft skills required to be a good developper. I'm taking this class to get experience working with other people, get the hang of git in a teamwork setting, and learn to communicate better.

    I want to work with other people on this thing I've mostly done on my own.

    My Concerns

    There are lots of things I don't know. There will always be lots of things I don't know. I'm the kind of person who likes to know everything I possibly can and knowing the sheer size of my ignorance is intimidating.

    My single biggest concern is that I've never worked on anything but my own projects and I'm worried I like learning to code more than I like coding.

    This card collects terms and their definitions into a Descritption List Element (dl) for later reference.

    tag name
    The word inside an HTML tag that defines the element.
    tag
    A tag name and opening and closing delimiters. A closing tag puts a forward slash in front of the tag name.
    Element
    An opening tag, some content, and a closing tag. Some elements don't have closing
    html
    it's a representational language, semantic markup, you use dumb things like article tags for google spiders
    article tag
    a unit of redistributable content
    user agent
    browser, usually. Actually I don't know what else it could be.
